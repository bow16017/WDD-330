  • Chapter 15: Modern Javascript - Modular JavaScript

    Here, I have decided to write my notes and definitions, as adding an example for this would be a little ridiculous.

    • Modular JavaScript refers to the use of 'self-contained' code (as the textbook phrases it) as a tool for multiple different projects or files.
    • A Module will contain functions, methods, etc. Each of these can be referred to again and again, so long as the host server has them imported.
    • I am used to seeing this kind of multi-tool reference in Java, where it is common to import a modular library to a project. This is similar, though not exact.
    • Many libraries of frameworks exist, each one offering a specific set of tools.
    • JavaScript modules will allow you access to new keywords (referenced through the library's methods), pre-built functions, and more.
    • Perhaps a precursor to this could be considered the age of javascript templates/freeware during the early 2000s. It is a common trend I've seen in websites built during that time: The website will use small 'snippets' or 'patches' of someone else's code in order to make their site run smoother. Now, of course, we have a much more integrated and MODULAR solution.
