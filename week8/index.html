CSS3 Transforms and Transitions

Transforms

The TRANSFORM property lets you:

  • Translate
    • Moves objects along the plane of your workspace.
  • Rotate
    • Rotates objects by point of origin. Specified by degree.
  • Scale
    • Resizes objects using horizontal and verical values. If only one value is provided, it is used for both values.
  • Skew
    • Skews an object using 'x' and 'y' plane values. If one of the values is not entered, a skew is not applied to that one, but is to the other.

It is possible to change the TRANSFORM property's origin. This is done using the TRANSFORM-ORIGIN property.

NOTE: The order that TRANSFORMs are applied to objects is important. Be sure the browser can digest the data in a way that makes logical sense and creates the desired effect.

Transitions

CSS TRANSITIONs present an easier and smoother way to animate that JavaScript. This is because it requires both less processing power, hard-coded sequences, and the ability to produce gradual animations.

Use TRANSITION-PROPERTY to specify which objects/properties are affected. Default property is set to 'all'.

Use TRANSITION-TIMING-FUNCTION to control the speed of animations in a very micro-focused way.

Use TRANSITION-DELAY to indroduce delays to the animation. This is important to create smooth and purposeful invocations by the user.

Multiple transitions can be applied and still work well, so long as they are neatly and efficiently constructed.

Animations

Animations are used to create more complicated, longer, and more strictly controlled movements.

They utilize the following:

  • Keyframes
  • Specific properits
    • name
    • duration
    • etc
  • Timing and iteration control
  • Color control

Canvas, SVG, and Drag and Drop

Canvas

The CANVAS API is used to draw images on a two dimensional plane via JavaScript.

A canvas object must first be created, and then drawing commands can be applied.

To command JavaScript to use a color, the following properties can be used:

  • strokeStyle
  • fillStyle
  • canvasGradient
  • canvasPattern

Shapes are drawn using the usual shape functions, with their accompanying values (radians, lengths, sides, etc).

Digital images, video, and text can be summoned within the canvas as well.

SVG

Vector-based imaging is a possibility and utility to be taken advantage of. Vector images are the best thing since pre-sliced bits... I am very used to using and creating them in almost all of my projects.

Using the SVG API is similar to CANVAS in that they both create and edit images, but the SVG takes longer because it is creating, editing, and displaying images that are responsive to digital manipulation.

One fantastic bonus to using SVG is that outside programs like Adobe Illustrator or Inkscape can be used to augment your program.

Drag and Drop

In order to make Drag and Drop features work, you must first make an item 'draggable'.

The DRAGGABLE value is not a boolean.

You must also create 'data transfer objects', which, in my own words, are like a buffer object which contains the data you wish to copy to the receptive field.

To make an object/field receptive, you must manage these two functions:

  • dragover
  • drop
